> 版本：v1.0 MVP | 日期：2026-04-18 | 状态：草稿

## 一、产品概述

### 1.1 产品定位

PhotoBook Studio 是一款面向个人用户的**画册/相册排版工具**，专注于将照片整理成精美的电子画册或可打印的 PDF 文件。产品定位为"轻量、快速、美观"，让任何人都能在几分钟内完成一本专业级画册。

### 1.2 核心价值主张

> **"把美好的回忆，变成捧在手里的画册。"**

- **零门槛**：无需设计基础，拖拖拽拽就能做画册
- **所见即所得**：实时预览，所见即所导出
- **快速交付**：从导入到导出，全流程不超过 5 分钟

### 1.3 目标用户

| 维度 | 描述 |
|------|------|
| 主要用户 | 25-45 岁，有旅行、家庭聚会、孩子成长记录等场景的个人用户 |
| 次要用户 | 小型工作室/摄影师，用于快速生成客户样片 |
| 技能水平 | 初级至中级，无需设计背景 |
| 使用频率 | 低频（按项目），但单次使用时长较长（30-60 分钟） |

## 三、功能详细说明

### 3.1 图片导入（F01）

**功能描述**

用户可以通过以下方式将照片导入应用：

1. **文件选择器**：点击"导入图片"按钮，打开系统文件选择器，支持多选 JPG/PNG/GIF/WebP/BMP 格式
2. **文件夹导入**：点击"导入文件夹"按钮，选择整个文件夹，自动筛选其中的图片文件
3. **拖拽导入**：将图片文件或文件夹拖入应用窗口，自动识别并导入

**验收标准**

- [ ] 支持同时选择多个文件（≥50张）
- [ ] 支持从文件夹批量导入（自动过滤非图片）
- [ ] 支持拖拽文件到窗口导入
- [ ] 导入完成后，图片显示在右侧缩略图库中
- [ ] 导入过程中显示进度状态

**技术说明**

- 图片以 DataURL（Base64）形式加载到内存，避免路径依赖
- 大图自动预缩放处理，避免内存溢出
- 建议使用 Worker 线程处理大量图片的解码

---

### 3.2 画布编辑（F02）

**功能描述**

基于 Fabric.js 引擎的交互式画布，支持：

1. **添加图片**：双击缩略图，将图片添加到画布中央
2. **选择对象**：单击画布上的对象显示选中框
3. **移动**：拖拽对象调整位置
4. **缩放**：拖拽控制点调整大小（保持比例）
5. **旋转**：拖拽旋转控制点旋转角度
6. **多选**：Shift+单击多选，批量操作

**验收标准**

- [ ] 双击缩略图，图片添加到画布
- [ ] 单击对象显示 8 点控制框和旋转控制点
- [ ] 拖拽对象可移动位置
- [ ] 拖拽控制点可缩放（保持宽高比）
- [ ] 拖拽旋转点可旋转
- [ ] Delete 键删除选中对象
- [ ] 缩放过程中画面流畅无卡顿

---

### 3.3 多页面管理（F03）

**功能描述**

画册由多个页面组成，每个页面独立编辑：

1. **添加页面**：点击"+"按钮添加空白页面
2. **删除页面**：点击"-"按钮删除当前页面（至少保留1页）
3. **切换页面**：点击左侧页面缩略图切换
4. **页面缩略图**：左侧面板实时显示所有页面的缩略图，当前页高亮

**验收标准**

- [ ] 至少保留 1 页，不能全部删除
- [ ] 页面缩略图实时更新（修改后自动刷新）
- [ ] 切换页面后画布内容正确切换
- [ ] 页面缩略图点击响应流畅

---

### 3.4 预设模板（F05）

**功能描述**

提供 6 种常用画册布局模板，一键应用到当前页面：

| 模板名称 | 布局说明 |
|----------|----------|
| 1 张图（全页） | 1张大图铺满页面，留5%边距 |
| 2 张图（上下） | 上下分布，各占一半 |
| 2 张图（左右） | 左右分布，各占一半 |
| 3 张图（1大2小） | 左侧1张大图，右侧上下2张小图 |
| 4 张图（2×2） | 2×2 网格均匀分布 |
| 网格（3×3） | 3×3 均匀网格 |

**验收标准**

- [ ] 每个模板在空白页上生成正确的布局区域
- [ ] 布局区域以虚线框标识
- [ ] 拖入图片时自动填充对应区域
- [ ] 模板选择后立即应用到当前页面

---

### 3.5 文字工具（F07）

**功能描述**

在画布上添加文字标注：

1. **添加文字**：点击"T"文字工具，在画布上添加文字框
2. **编辑内容**：双击文字框进入编辑模式
3. **样式控制**：支持字体、字号、颜色、加粗、斜体、下划线
4. **对齐控制**：左对齐/居中/右对齐

**验收标准**

- [ ] 点击文字工具后在画布上添加文字框
- [ ] 双击文字进入编辑模式
- [ ] 文字属性（字体/字号/颜色/加粗/斜体/下划线）实时生效
- [ ] 文字对齐方式正确应用

---

### 3.6 边框与阴影（F09, F10）

**功能描述**

为画布上的图片添加装饰效果：

**边框**
- 颜色：颜色选择器
- 宽度：0-50px 可调

**阴影**
- 开关：启用/关闭
- 水平偏移：-50 到 +50px
- 垂直偏移：-50 到 +50px
- 模糊度：0 到 50px
- 颜色：颜色选择器

**验收标准**

- [ ] 边框颜色和宽度实时预览
- [ ] 阴影各项参数实时预览
- [ ] 参数调整后无需点击确认，立即生效

---

### 3.7 导出 PDF（F04）

**功能描述**

将整个画册导出为 PDF 文件：

1. 点击"导出 PDF"按钮
2. 弹出系统保存对话框，默认文件名带时间戳
3. 逐页渲染画布内容，合并为多页 PDF
4. 完成后显示保存路径

**验收标准**

- [ ] 导出 PDF 文件可被标准 PDF 阅读器打开
- [ ] 多页画册正确生成为多页 PDF
- [ ] 图片质量清晰（使用高分辨率渲染）
- [ ] 文件保存路径可自定义
- [ ] 导出过程显示进度

**技术说明**

- 使用 jsPDF + html2canvas 方案
- 每页作为独立图片嵌入 PDF
- JPEG 质量设为 95%

---

### 3.8 撤销/重做（F06）

**功能描述**

支持操作历史的撤销和重做：

- **撤销**：Ctrl+Z，回退上一步操作
- **重做**：Ctrl+Y，恢复被撤销的操作
- 历史栈深度：最多保留 50 步

**验收标准**

- [ ] Ctrl+Z 撤销最近一次操作
- [ ] Ctrl+Y 重做被撤销的操作
- [ ] 新操作清空重做栈
- [ ] 工具栏撤销/重做按钮状态正确

## 四、非功能性需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 启动时间 | ≤ 5 秒（首次启动） |
| 图片导入（10张） | ≤ 3 秒 |
| 页面切换响应 | ≤ 100ms |
| 导出 PDF（10页） | ≤ 30 秒 |
| 内存占用 | 空闲时 ≤ 300MB，50张图片时 ≤ 1GB |

### 4.2 兼容性要求

- **操作系统**：Windows 10 及以上（64位）
- **屏幕分辨率**：1280×720 及以上
- **内存**：推荐 8GB 及以上

### 4.3 UI/UX 要求

**视觉风格**

- 深色主题，减少视觉疲劳
- 主色调：Indigo（#6366f1）
- 字体：系统默认无衬线字体

**布局规范**

- 左侧面板：160px，页面缩略图列表
- 中央画布：自适应，可缩放显示
- 右侧面板：220px，功能面板（图片库+属性）
- 顶部工具栏：48px，工具和操作按钮

**交互反馈**

- 所有按钮有 Hover 和 Active 状态
- 操作结果在底部状态栏显示
- 加载过程显示 Loading 遮罩

### 4.4 数据持久化

MVP 阶段**不包含**自动保存功能，用户需手动导出保存。建议未来版本增加：

- 项目文件保存/加载（.pbs 格式）
- 自动保存草稿（每 2 分钟）

## 七、风险与依赖

| 风险/依赖 | 影响 | 缓解措施 |
|-----------|------|----------|
| Fabric.js CDN 访问不稳定 | 离线无法使用 | 考虑将库打包到本地 |
| PDF 导出内存溢出（大图） | 应用崩溃 | 增加图片尺寸限制，分块渲染 |
| Electron 打包体积过大 | 用户体验差 | 使用 Tauri 替代 Electron（未来） |
| Windows Defender 误报 | 用户不信任 | 申请代码签名（商业化阶段） |

## 八、附录

### 8.1 术语表

| 术语 | 说明 |
|------|------|
| MVP | Minimum Viable Product，最小可行产品 |
| 画布（Canvas） | 编辑区域，显示当前页面内容 |
| 页面（Page） | 画册的一页，包含多个对象 |
| 对象（Object） | 画布上的图片或文字 |
| Fabric.js | Web 画布交互库 |
| jsPDF | JavaScript PDF 生成库 |
| Electron | 跨平台桌面应用框架 |

### 8.2 参考竞品

- **Canva**：功能强大但复杂，适合通用设计
- **Photobook Builder**：传统软件，上手稍难
- **Mixbook**：在线工具，依赖网络

> 本产品定位更轻量，聚焦"快速出结果"，差异化在于零学习成本。
